Highlight #1 What is Version Control?

Version control is a safety net. It is all about change and change over time. With version control you can go and recall files before they were modified as well as after modifications. Version control is very helpful for web designers in case they make mistakes or need to go back and change something. You can also see what might be causing a problem for your webpage and who caused it in the first place.

Highlight #2 How Does Git Work?

Git plays its role by taking snapshots of what your files look like. Every time you type the git commit command, a snapshot is taken and stored. This is also important for web designing because you can go back and view the snapshots and see what is changed. If nothing has changed and git commit is typed in, git wonâ€™t store another copy but just keep the original. Git is good for storing files because those usually will not get lost compared to using other VCS systems. There are three stages of git. There is committed, modified, and staged. Committed means the data is stored, modified means you are working on the file but have got typed git commit yet, and staged is when you have yet to modify the file.

Highlight #3 Repositories

A repository is where all the files that have been changed are stored so you can go back and retrieve them. The repository is where all the commit objects go. When you type git commit the files are sent to the repository. We created repositories on GitHub that we can go back and retrieve our files to see what we have changed and where we may have made a mistake.
